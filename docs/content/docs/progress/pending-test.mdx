## PPT 内容合同教学法重构 SHA-30

- **初稿自动整理**：让模型生成一份封面带重复正文/词表外排版词（如「瀑布式」）的内容方案，入库后应看不到对应报错，页面卡片出现「已自动整理 N 处」可展开留痕。
- **理念问题卡**：封面带真实正文内容时，「内容检查」不再出现判决句，改为「理念问题」卡（正面陈述 + 移到下一页/删除该块/保留——我要这样）；选「保留」后仪表显示「按你的设计保留」且可撤销；选「移到下一页」块随来源迁到第 2 页。
- **偏离承接**：保留偏离后确认内容并生成，该页最终提示词应含承接指令（如「封面保持定位语视觉主导…补充内容极轻量」）。
- **职责仪表**：每页卡片 purpose 下方有仪表行（封面：定位语/无多余承载；内容页：核心信息/支撑 N 条/来源齐；收尾：收束信息/无新增正文），正常态绿勾、待办中性圆点,无红色。
- **来源聚合**：同一材料区间被多个块引用时,「查看 N 条来源依据」按区间归并并标「支撑 N 个内容块」。
- **布局词表**：「上下主次/居中聚焦/步骤堆叠」等纯排版描述不再被报「无法识别」;含数字/文案的排版要求仍被拦截。
- **验证边界**：自动化 322 用例 + typecheck 已绿,双主题浏览器交互验收已过(注入草稿实测);真实 LLM 生成链路(初稿→整理→问题卡)待用户线上跑一轮确认。

## PPT 逐页外壳事实与参考图约束 SHA-29

- **页码与外壳**：多页确认后「生成其余页」时，每页最终提示词应含正确 `本页页码 N / 总页数 M`（M 为整套总页数，不是本批张数）、整套标题；有章节页时内容页页眉章节标签随最近 section 变化，封面/收尾不标章节；无章节结构时明确「本套无章节分组」。
- **参考图不钉死版式**：以风格校样作参考生成其余页时，页眉文字、页码、目录/侧栏结构不得照搬校样图；正文构图应随本页 contentForm 变化，不机械重复。
- **shell 全 none**：页眉页脚均为 none 时不注入「本页页面事实」与参考图角色约束段。
- **验证边界**：自动化 `ppt-prompt-compiler` / `ppt-style-proof-flow`（含 `assertGenerationPlanCompilation` rebuild）与 typecheck 已绿；真实生图验收由用户确认，调用前需单独确认预算。

## PPT 内容来源与归纳 Contract SHA-26

- **derived 归纳**：模型对材料压缩/改写并声明 `relation=derived` 且行号合法、数字/大写术语落在引用行时，页面只显示一次正文，来源为「基于材料/补充要求 Lx–Ly 归纳」，不出现「引用范围无效」、同文 AI 建议，也不形成 blocking gap。
- **verbatim 重绑**：逐字内容仍严格匹配；错误行号但原文存在时自动重绑为 `relation=verbatim`。
- **新增硬事实**：derived 引入材料未支持的数字/大写术语时仍阻断，问题为短句（如「请确认本页核心信息中的新增表述」）；若建议与页面已显示正文相同，不渲染第二份「AI 建议」，提供「确认采用当前内容」。
- **Deck Brief**：受众/目标/叙事主线是作者侧归纳；字段非空即可进入草稿，空字段才产生 missing_detail。
- **验证边界**：`node --test tests/ppt-content-plan.test.mjs tests/ppt-content-plan-step.test.mjs tests/ppt-content-request.test.mjs`；localhost 仅用假 Provider，真实模型调用为 0。

## PPT 信息缺口与来源状态 SHA-22 / SHA-23 / SHA-24 / SHA-25

- **来源重绑（SHA-22 / SHA-24）**：正文完整存在于原材料但模型行号错误或缺失时，应自动绑回最小支持行范围；不得再生成与原文相同的「请确认或补充」及 AI 建议。
- **空字段问题（SHA-23）**：空核心信息只出现「请补充本页核心信息」类字段级问题，不得出现「请确认或补充：本页内容」。
- **纯占位（SHA-25）**：列表等为「待补充 / 请补充」时归一为占位块 + 具体 missing_detail；同一块已有未解决信息缺口时，来源区不再并列「来源待确认」。
- **状态分区**：继续只用「信息缺口 / 已确认补充」两区，不新增第三套状态列表；真不在材料中的 AI 新文案仍需确认。
- **验证边界**：`node --test tests/ppt-content-plan.test.mjs tests/ppt-content-plan-step.test.mjs`；无真实模型调用。

## PPT 单页修复闭环 SHA-19 / SHA-20 / SHA-21

- **超长文案（SHA-19）**：内容方案出现「单页文案过长」时，按钮文案为「压缩本页」；假 Provider 返回同长文案应失败并保留原页，返回压缩后文案应成功且该问题消失。
- **可见终态（SHA-20）**：页级修复过程中显示「正在修复本页…」且相关按钮 loading/禁用；成功提示「本页已更新」；失败提示具体未解决问题且原页不变。
- **封面职责（SHA-21）**：封面检查失败时按钮为「修复封面」；不合格封面结果不得替换成功；合格封面（标题 + 一句定位语、无正文块）可替换并清除 invalid_cover。
- **通用补全**：页头「让 AI 补全本页」仍走同一 regenerate 管线，不另开第二套生成路径。
- **验证边界**：`node --test tests/ppt-content-plan.test.mjs tests/ppt-content-plan-step.test.mjs`；localhost 仅用假 Provider，真实模型调用为 0。

## PPT 创建草稿恢复 SHA-18

- **列表可发现**：有未完成草稿时，`/ppt` 列表主按钮文案为「继续创建」，并有「有未完成的创建草稿」提示；无草稿时仍为「新建 PPT」。不得用「新建 PPT」静默打开未知草稿。
- **离开再进**：在 `/ppt` 向导中粘贴材料并生成分页（extract 或 outline 内容方案均可），返回列表后再点「继续创建」；材料、步骤、分页/内容方案/`contentDraft` 应仍在，并提示「已恢复上次未完成内容」。
- **整页刷新**：在向导有已提交进度时刷新浏览器，再打开流程，恢复结果与路由往返一致；不得出现假 loading，也不得自动再打文本/视觉模型（假 Provider 请求计数保持 0）。
- **视觉步骤**：在 outline 路径进入视觉方向并已有 Contract 后离开再恢复，应直接显示已恢复 Contract 与 PageSpecs，不得自动重新请求视觉候选。
- **重新开始**：点「重新开始」并确认后草稿清除、向导回到初始；清除失败时不得假重置界面，应提示失败。再次进入不再恢复旧内容。
- **建图清除**：成功「生成画布」后返回 `/ppt`，不应再把已完成工程当成未完成草稿。若工程已创建但草稿清理失败，应提示清理失败（不是「建图失败」），并正常进入画布，不诱导重复建图。
- **验证边界**：聚焦测试 `node --test tests/ppt-creation-resume.test.mjs`；localhost 冒烟仅用假 Provider，文本/文生图/图生图真实调用均为 0。

## PPT 内容规划 SHA-17 修复

- **创作目标不再上屏**：用包含“我想做一份……”“这份材料要让……理解……”的材料生成内容方案；这些句子应进入整套 Brief，不再成为标题、核心信息或正文。正式章节标题（如“为什么需要 PPT 工作台”）和普通业务主张仍应保留。
- **封面与叙事**：多页方案第一页必须是封面，只保留标题和一句定位语，不出现目标清单或长正文；第二页直接进入产品问题或价值。用户明确写“最多 N 页 / N 页以内”时不得超限，未写页数时不套用固定上限。
- **布局审核与退出路径**：`居中主视觉 · 上下分区`、`四宫格或纵向编号列表`、`2×2 网格`等纯几何表达应直接通过；夹带未批准厂商、数字或事实时应点名原布局，并可预览后只移除该布局要求，不修改正文、来源或已确认事实，也不调用模型。
- **单页重生成闭环**：从审核问题重生成本页时，请求应携带具体问题；若新结果仍有同一问题，原页必须保留并显示“问题仍未解决”。其他页面、已确认正文和 gap lineage 不变。
- **缺口状态**：“信息缺口”和“需要你决定”只统计未解决项；已采用用户补充、已确认建议、占位或不呈现决定保留在默认折叠的“已确认补充”，不再看起来仍待处理。
- **验证边界**：自动化、typecheck、任务相关格式检查、build 与 localhost 冒烟均使用固定夹具或本机假 Provider；文本、文生图、图生图真实模型调用均为 0。

## PPT 精修与工作区体验

- **修改入口**：从候选稿进入“按要求改图”，分别验证仅整页要求、仅一个或多个点位、整页要求 + 多点位三种组合；清除文字和清除点位互不影响，提交前的修改清单、最终提示词和确认内容完全一致。零点位时不得出现红色标记或“所指对象”等话术。
- **固定单张**：先把方案生成数设为 3，再分别执行“再生成一稿”和“生成修改稿”；两者都应明确为 1 张 / 1 次调用，只追加一个候选，方案原有的 3 张配置不变。“再生成一稿”不带修改要求。
- **原稿与定稿保护**：两种精修动作都不覆盖基图、不改写已确认最终版；修改稿生成中仍能查看基图，界面明确显示“正在基于此稿生成修改稿”，并且可切换页面继续工作。
- **候选追溯与定位**：修改稿刷新后仍能查到所属 `pageId` / `takeId`、基图、整页要求、点位快照、最终提示词、`runId` 和 `requestId`；点击完成通知时按 `runId` 选中该轮真实候选，旧 Run 晚完成也不得定位到新 Run 的结果。
- **上下区域调节**：在中间栏上方方案说明区与下方候选区之间上下拖动分隔线，确认两区都保留最小可操作高度；切换页面或方案再返回时，已选候选和该方案的滚动位置保持。
- **尺寸、主题与回归**：在 1512×700、1512×785 与浅色 / 深色主题下确认主图、候选、按钮和分隔线无溢出；同时回归新建方案的大提示词编辑区、最终检视的页码 / 方案 / 候选序号 / 确认状态，以及返回结构画布后的拖动、缩放、选中和点击。
- **验证边界**：本节可使用静态图片和假任务完成自动化、typecheck、build 与 localhost 验收；未经单独确认调用预算时，文本、文生图和图生图真实模型调用均必须为 0。

## PPT 生成可靠性

- **稳定身份**：新建 PPT 后新增、复制、切换和删除方案；页面排序与节点引用变化时，页/方案选择、确认图和候选归属仍保持正确，不再用页序或配置节点 ID 充当业务身份。
- **保存后再请求**：批量与单页生成都先写入结构和固定请求槽，经 localforage 读回并确认 `submitting` 锁后才发请求；落盘或读回失败时不得产生 POST。确认后到 POST 前替换共享参考图时，请求仍使用确认瞬间冻结的图片快照。
- **多图请求槽**：分别验证 count=1 与 count=3。count=1 的同一节点同时持运行摘要和请求 trace；count=3 为一个 root + 三个固定子槽，每槽最多一次 POST，成功后在原槽变成候选；其他槽仍更新进度时，用户已选择的主图和展开状态不能被旧持久快照覆盖。
- **刷新恢复**：异步任务处于 submitted/running、远端成功但尚未保存、结果保存失败三个阶段时刷新，只继续查询原 task ID，不新增 POST、不重复候选；通用画布恢复 hook 不得同时接管 PPT 请求槽。
- **原渠道恢复**：请求槽冻结 channel ID、地址、协议和模型，但不保存 API Key。任务运行后删除渠道、改地址/协议或移除模型，重新获取应停在需处理状态并要求恢复原配置，不得默认切到其他渠道。
- **未知提交**：在 durable `submitting` 后、task ID 落盘前中断，重开后显示“提交结果未知”且不自动重提；可标记放弃。此时再次生成必须明确提示可能重复计费，并创建全新的 batch/run/request。
- **重新生成预算**：已有历史 Run/候选时，普通重新生成再次显示新请求数与文生图/图生图构成；未知请求标记放弃后，单页和批量入口仍保留重复计费警告，不得因状态已变成 `abandoned` 绕过确认。
- **按要求改图**：PPT 候选生成修改稿前确认 1 次图生图，并创建新的 batch/run/request；刷新恢复、终态候选与诊断均走同一模块，不再产生无台账子节点。
- **终态与通知**：完成、部分完成、失败和需要处理的应用内提示每个 run/终态只出现一次，点击能回到对应页；刷新后不重复提示。工作台“生成记录”按 batch 展示跨页每个 Run，并可展开 Request、task ID、预计/返回/完成数量和耗时。错误区可复制不含提示词、图片正文、API Key 的结构化诊断。
- **错误分类**：无 task ID 的 5xx/408/断网按提交结果未知处理；已有 task ID 的临时轮询或取图错误保持可重新获取；明确远端终态、可确认未执行的 4xx 拒绝或已知 0 POST 的本地准备失败显示中性的“已失败”，并保留具体原因。远端已成功但图片过期或缺失时，还要持久化重复计费风险，之后的单页、批量、派生方案和按要求改图均必须再次警告。
- **台账保护与回归边界**：普通画布与 Agent 的非 PPT `run_generation` 保持原行为；PPT 归属节点不能通过结构画布生成/重试、上传替换、输入改写、删连线、撤销/重做或普通图片派生绕过请求槽。未解决请求存在时，从“我的画布”“我的 PPT”和工程内都不能删除工程；返回结构画布后仍要实际拖动、缩放、选中和点击，确认没有不可见弹层拦截。
- **验证边界**：本轮自动化可靠性测试 14/14、typecheck、build 与 localhost 本机假接口真实路径冒烟均通过；localhost 仅向本机假接口发起 1 次文本分页请求，图片请求为 0。真实模型调用保持 0；建议人工真实复测上限仍为文本 0、文生图 2、图生图 1，执行前需再次确认模型、单价与费用。

## PPT 工作台第三批

> Agent 已完成受控真实调用冒烟：文本改写 1 次、文生图 2 次、图生图 2 次；均成功且无重试，两个 `count=2` 路径都回填 2 个候选稿，刷新后结果仍保留。以下条目继续留给用户做最终体验确认。

- **方案与候选**：切换器在 5 个以上方案时正确收纳且当前方案常驻；每次只显示当前方案候选；历史批量根确认可解析到主图；删除方案会清理根、全部子图与失败产物，生成中禁止删除。
- **方案删除后的撤销与存储**：在专用测试工程删除带图片候选的方案后，画布节点、连线、确认状态和对应图片 Blob 均清理；点击普通画布「撤销」不得复活已删方案的孤儿节点，其他普通画布图片删除仍可撤销恢复。
- **生成配置与预算**：摘要显示 effective 模型、尺寸、张数；修改后画布配置面板与下一次生成一致；`count=2` 时两张成功子图均为候选，按钮和批量确认显示真实 2 次调用；缺配置页明确跳过；修复后再次「生成其余」仍补首页参考图并计为图生图。
- **提示词大编辑器**：长提示词固定高度内滚；1512×700 视口下弹窗与底部按钮不溢出；取消不改节点；已有候选的方案只提供「保存为新方案 / 保存并生成新方案」；AI 改写可取消、手改会使迟到结果失效、失败保留原稿、一次撤销可用。真实 AI 改写需先确认文本调用预算。
- **生成配置浮层**：首次点击摘要即可看到模型、尺寸、张数和「保存配置」，切换方案时浮层关闭且不带入上一方案草稿；修改后仍需确认画布配置节点和下一次生成使用同一值。
- **空间与回归**：1512px 宽下按 700/785px 两个高度边界实测精修台与最终检视主图；浅/暗主题均无溢出；返回结构画布后实际拖动、缩放、选中和点击节点，确认无透明弹层拦截。
