## PPT Prompt Compiler

- **统一规格模型**：分别用“从原始材料规划”和“已有逐页规格”建稿，确认两种入口都落到同一套 DeckBrief、PageSpec 和后续工作台；原材料模式对无法溯源的结论要求确认，规格直出模式保持页数、顺序和原文切片。
- **正文与布局保真**：用 8 页样本核对页序、锁定正文、数字、术语、5 点列表、整页左对齐和双柱状图布局；删除、重排、增加第 6 点或加入未经确认的结论时必须阻断生成。
- **最终提示词可见**：单页和批量确认均展示真正提交的完整提示词、来源、检查结果、页数与文生图/图生图调用数；未修改的快速生成与展开审阅必须使用同一编译结果。
- **两类编辑**：调整页面规格会更新 PageSpec；直接调整最终提示词保存为显式 override。新增、删减、改写、重排或改变段落归属后都必须重新确认；自定义排版文本同样只对完全一致的确认版本生效。
- **禁止项与提交绑定**：后续编辑风格时，新增或删除的“禁止…”规则要同步进入 DeckBrief，并保留需求中的禁止项；冲突时不得 POST。每个 Request 必须绑定不可变 CompilationSnapshot，快照缺失、被篡改、输入过期或 PageSpec 升版时均在请求前失败。
- **结构画布保护**：PPT 大纲、风格、来源、生成配置、候选和 Run/Request 归属只由工作台修改；结构画布仍可拖动、缩放、选中、查看和下载，但不能经通用编辑、生成、重试、替换、删除、连接、撤销、Agent 或图像派生工具绕过工作台。
- **验证边界**：自动化测试 58/58、typecheck、生产构建与 localhost 本机真实交互均通过；浏览器覆盖最终提示词、显式确认、批量预算、刷新持久化及结构画布拖动/缩放/选中/保护。本轮文本、文生图、图生图真实模型调用均为 0，因此尚未验证真实模型的图片内容与排版质量。

## PPT 生成可靠性

- **稳定身份**：新建 PPT 后新增、复制、切换和删除方案；页面排序与节点引用变化时，页/方案选择、确认图和候选归属仍保持正确，不再用页序或配置节点 ID 充当业务身份。
- **保存后再请求**：批量与单页生成都先写入结构和固定请求槽，经 localforage 读回并确认 `submitting` 锁后才发请求；落盘或读回失败时不得产生 POST。确认后到 POST 前替换共享参考图时，请求仍使用确认瞬间冻结的图片快照。
- **多图请求槽**：分别验证 count=1 与 count=3。count=1 的同一节点同时持运行摘要和请求 trace；count=3 为一个 root + 三个固定子槽，每槽最多一次 POST，成功后在原槽变成候选；其他槽仍更新进度时，用户已选择的主图和展开状态不能被旧持久快照覆盖。
- **刷新恢复**：异步任务处于 submitted/running、远端成功但尚未保存、结果保存失败三个阶段时刷新，只继续查询原 task ID，不新增 POST、不重复候选；通用画布恢复 hook 不得同时接管 PPT 请求槽。
- **原渠道恢复**：请求槽冻结 channel ID、地址、协议和模型，但不保存 API Key。任务运行后删除渠道、改地址/协议或移除模型，重新获取应停在需处理状态并要求恢复原配置，不得默认切到其他渠道。
- **未知提交**：在 durable `submitting` 后、task ID 落盘前中断，重开后显示“提交结果未知”且不自动重提；可标记放弃。此时再次生成必须明确提示可能重复计费，并创建全新的 batch/run/request。
- **重新生成预算**：已有历史 Run/候选时，普通重新生成再次显示新请求数与文生图/图生图构成；未知请求标记放弃后，单页和批量入口仍保留重复计费警告，不得因状态已变成 `abandoned` 绕过确认。
- **标注改图**：PPT 候选的标注改图在 POST 前确认 1 次图生图，并创建新的 batch/run/request；刷新恢复、终态候选与诊断均走同一模块，不再产生无台账子节点。
- **终态与通知**：完成、部分完成、失败和需要处理的应用内提示每个 run/终态只出现一次，点击能回到对应页；刷新后不重复提示。工作台“生成记录”按 batch 展示跨页每个 Run，并可展开 Request、task ID、预计/返回/完成数量和耗时。错误区可复制不含提示词、图片正文、API Key 的结构化诊断。
- **错误分类**：无 task ID 的 5xx/408/断网按提交结果未知处理；已有 task ID 的临时轮询或取图错误保持可重新获取；明确远端终态、可确认未执行的 4xx 拒绝或已知 0 POST 的本地准备失败显示中性的“已失败”，并保留具体原因。远端已成功但图片过期或缺失时，还要持久化重复计费风险，之后的单页、批量、派生方案和标注改图均必须再次警告。
- **台账保护与回归边界**：普通画布与 Agent 的非 PPT `run_generation` 保持原行为；PPT 归属节点不能通过结构画布生成/重试、上传替换、输入改写、删连线、撤销/重做或普通图片派生绕过请求槽。未解决请求存在时，从“我的画布”“我的 PPT”和工程内都不能删除工程；返回结构画布后仍要实际拖动、缩放、选中和点击，确认没有不可见弹层拦截。
- **验证边界**：本轮自动化可靠性测试 14/14、typecheck、build 与 localhost 本机假接口真实路径冒烟均通过；localhost 仅向本机假接口发起 1 次文本分页请求，图片请求为 0。真实模型调用保持 0；建议人工真实复测上限仍为文本 0、文生图 2、图生图 1，执行前需再次确认模型、单价与费用。

## PPT 工作台第三批

> Agent 已完成受控真实调用冒烟：文本改写 1 次、文生图 2 次、图生图 2 次；均成功且无重试，两个 `count=2` 路径都回填 2 个候选稿，刷新后结果仍保留。以下条目继续留给用户做最终体验确认。

- **方案与候选**：切换器在 5 个以上方案时正确收纳且当前方案常驻；每次只显示当前方案候选；历史批量根确认可解析到主图；删除方案会清理根、全部子图与失败产物，生成中禁止删除。
- **方案删除后的撤销与存储**：在专用测试工程删除带图片候选的方案后，画布节点、连线、确认状态和对应图片 Blob 均清理；点击普通画布「撤销」不得复活已删方案的孤儿节点，其他普通画布图片删除仍可撤销恢复。
- **生成配置与预算**：摘要显示 effective 模型、尺寸、张数；修改后画布配置面板与下一次生成一致；`count=2` 时两张成功子图均为候选，按钮和批量确认显示真实 2 次调用；缺配置页明确跳过；修复后再次「生成其余」仍补首页参考图并计为图生图。
- **提示词大编辑器**：长提示词固定高度内滚；1512×700 视口下弹窗与底部按钮不溢出；取消不改节点；已有候选的方案只提供「保存为新方案 / 保存并生成新方案」；AI 改写可取消、手改会使迟到结果失效、失败保留原稿、一次撤销可用。真实 AI 改写需先确认文本调用预算。
- **生成配置浮层**：首次点击摘要即可看到模型、尺寸、张数和「保存配置」，切换方案时浮层关闭且不带入上一方案草稿；修改后仍需确认画布配置节点和下一次生成使用同一值。
- **空间与回归**：1512px 宽下按 700/785px 两个高度边界实测精修台与最终检视主图；浅/暗主题均无溢出；返回结构画布后实际拖动、缩放、选中和点击节点，确认无透明弹层拦截。
