# PPT Workbench Context

本文定义 PPT 场景叠加在通用画布之上的领域语言。界面文案可以保留已有别名，但需求、设计和后续代码命名应优先使用这里的正式术语。

## Language

**PPT 工程**

一套待交付演示文稿的聚合根，包含有序页面、共享素材与风格、逐页确认和最终交付状态。
_Avoid_: 在讨论 PPT 业务时用“画布工程”代替它。

**页面**

PPT 工程中按交付顺序排列的单位；拥有多条方案分支，并在交付前指向一个最终候选稿。
_Avoid_: 用节点行、画布区域或图片节点代表页面本身。

**方案分支**

某一页面下保留的一套独立方案，包含一份稳定的分支提示词和由它生成的候选稿。界面可继续显示“线路 1 / 线路 2”。
_Avoid_: 把同一分支的提示词原地改写后继续生成，导致历史方案失真。

**分支提示词**

驱动一个方案分支生成的页面规格。分支尚无候选稿时可以编辑；已有候选后，通过“基于此方案调整”派生新分支。
_Avoid_: 把页面原始大纲、运行时拼装后的完整 prompt 和分支提示词混为同一个概念。

**候选稿**

某个方案分支下的一次可选择视觉结果，界面可以显示为 V1、V2。
_Avoid_: 单独使用含义过宽的“版本”作为领域名。

**页面最终确认**

页面级选择，指向该页某一方案分支中的一个有效候选稿；它不属于方案分支。
_Avoid_: 用“当前查看”“最新生成”或某条分支的活动状态隐式代替最终确认。

**PPT 工作台**

围绕页面、方案分支、候选稿、标注、确认和交付组织的任务型主界面。
_Avoid_: 把它描述成画布上的临时辅助面板。

**结构画布**

同一个 PPT 工程的底层节点与连线视图，用于理解结构、定位异常和高级编辑。
_Avoid_: 要求普通 PPT 精修流程必须依赖节点的全局几何位置完成。
