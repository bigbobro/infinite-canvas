# PRD：PPT 工作台 P0（向导建图 + 批量生成 + 确认下载闭环）

## 背景

在现有生图能力之上新增「PPT 工作台」模块。完整设计已定稿：`docs/content/docs/development/ppt-workbench-design.mdx`（下称「设计文档」），本任务实现其 §9 定义的 P0 范围。

## 需求（P0 范围）

用户视角的完整闭环：

1. **材料输入**：在 `/ppt` 向导中粘贴文字材料（Markdown 或整文），填写 PPT 要求。
2. **分页大纲**：系统调用文本 LLM 生成分页大纲（页数、每页标题、要点、视觉建议）；**用户可在向导内编辑**（改标题/要点、增删页）后确认。
3. **风格配置**：确认大纲后配置风格——风格描述文本 +（可选）参考图上传。
4. **建图落画布**：确认后自动创建一个专属画布工程（一 deck = 一工程：风格区节点 + 每页「大纲文本 → 生成配置」节点列），跳转到该画布。
5. **批量生成**：画布内 PPT 面板提供「全部生成」（默认首页锚定两段式：先第 1 页 → 确认后其余页以它为参考批量生成；可跳过直接全出）与「单页重新生成」。
6. **确认与下载**：面板对每页可「确认此页」（记录最终图像节点）；全部处理后「打包下载」导出按页序命名的图片 zip（`01_<页标题>.png` …）。

## 明确不做（超出即越界）

- ❌ `.pptx` 生成（用户明确拍板：只做图片）。
- ❌ 任何后端/服务端能力（纯前端项目）。
- ❌ P1/P2 项：风格预设库、并发队列、Agent 站点工具、WebDAV 同步 domain、讲者备注。
- ❌ 精修能力的任何新实现——多轮精修完全复用画布既有功能（自身参考再生成、蒙版重绘等），面板只追踪确认指针。

## 约束（已拍板决策）

- 遵守薄接缝原则：新代码全部在新文件；上游文件只做侵入点清单（设计文档 §7.4）列明的 6 处追加式修改；如实际侵入与清单不符，必须同步更新清单。
- 首页锚定默认开、可跳过。
- 大纲编辑为轻量版（标题/要点/增删页），不做每页版式类型等重型结构化编辑。
- 开发分支 `feat/ppt-workbench-p0`，提交前缀 `feat(ppt): …`。

## 验收标准

- [ ] `cd web && npm run typecheck` 通过。
- [ ] `/ppt` 进入向导，完整走通：材料 → 大纲生成（流式可见）→ 编辑大纲 → 风格配置 → 建图跳转画布。
- [ ] 建出的画布工程结构正确：风格节点连入每页 config，页大纲文本连入对应 config，连线语义能被既有生成管线消费（有参考图时走图生图）。
- [ ] PPT 面板五个动作可用：页状态总览 / 全部生成（含首页锚定与跳过）/ 单页重生成 / 确认此页 / 打包下载。
- [ ] 单页生成失败不影响其他页，可单独重试。
- [ ] zip 内文件按页序命名且为各页确认图（未确认页取该页最新成功图）。
- [ ] 非 PPT 画布工程行为无任何变化（防回归）。
- [ ] 文档同步：CHANGELOG `Unreleased` 追加、`todo/pending-test` 按 `.trellis/spec/project/docs-and-release.md` 流水线更新。
